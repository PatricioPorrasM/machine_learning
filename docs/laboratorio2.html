2. Preprocesamiento de datos

Creaci√≥n de la Variable Objetivo (Target)

Justificaci√≥n del Target:

Para este laboratorio, el objetivo es clasificar si un tweet proviene de un usuario ‚Äúinfluyente‚Äù o ‚Äúcom√∫n‚Äù. Esta es una tarea de clasificaci√≥n binaria muy √∫til para an√°lisis de redes sociales.

Definiremos un usuario ‚Äúinfluyente‚Äù bas√°ndonos en dos criterios claros y com√∫nmente aceptados:

  1. Verificaci√≥n de la cuenta (authorVerified): Twitter verifica cuentas de inter√©s p√∫blico, lo que es un fuerte indicador de influencia.
  2. N√∫mero de seguidores (authorFollowers): Un alto n√∫mero de seguidores denota un gran alcance.

Regla de Negocio: Un usuario ser√° considerado influyente (clase 1) si su cuenta est√° verificada O si tiene m√°s de 50,000 seguidores. Este umbral es una heur√≠stica com√∫n para diferenciar cuentas con un alcance masivo. De lo contrario, ser√° clasificado como com√∫n (clase 0).

# Definir el umbral de seguidores
follower_threshold = 50000

# Crear la columna target 'is_influencer'
# Ser√° 1 (influyente) si la cuenta est√° verificada O tiene m√°s seguidores que el umbral
# Ser√° 0 (com√∫n) en caso contrario
df['is_influencer'] = ((df['authorVerified'] == True) | (df['authorFollowers'] > follower_threshold)).astype(int)

4. Entrenamiento del modelo

Pipeline de Preprocesamiento y Modelado

Para manejar eficientemente los diferentes tipos de datos (texto, num√©ricos, categ√≥ricos), creamos un Pipeline que integra el preprocesamiento y el modelo.

Estrategia de preprocesamiento:

  1. Texto (content, hashtags_cleaned): Se vectorizar√°n usando TfidfVectorizer, que convierte el texto en una matriz de features num√©ricos basada en la frecuencia e importancia de cada palabra.
  2. Num√©ricos: Se escalar√°n con StandardScaler para que tengan media 0 y desviaci√≥n est√°ndar 1, evitando que variables con escalas grandes dominen el modelo.
  3. Categ√≥ricos: Se codificar√°n con OneHotEncoder, que crea una columna binaria para cada categor√≠a.
# Definir las columnas para cada tipo de preprocesamiento
text_features = ['content', 'hashtags_cleaned']
numeric_features = ['account_age_days', 'mentions_count', 'hashtags_count', 'content_length']
categorical_features = ['isReply', 'has_profile_picture']

# Crear el transformador de preprocesamiento
preprocessor = ColumnTransformer(
    transformers=[
        ('content_tfidf', vectorizer_content, 'content'),
        ('hashtags_tfidf', vectorizer_hashtags, 'hashtags_cleaned'),
        ('numeric', StandardScaler(), numeric_features),
        ('categorical', OneHotEncoder(handle_unknown='ignore'), categorical_features)
    ],
    remainder='passthrough' # Mantener otras columnas si las hubiera (no aplica aqu√≠)
)

# Crear el pipeline final que une el preprocesador y el modelo
model = Pipeline(steps=[
    ('preprocessor', preprocessor),
    ('classifier', LogisticRegression(random_state=42, class_weight='balanced', max_iter=1000))
])

# Usamos class_weight='balanced' para que el modelo preste m√°s atenci√≥n a la clase minoritaria (influyentes).

7. Conclusiones y Reflexiones

¬øC√≥mo se seleccion√≥ y justific√≥ el target?

Se cre√≥ una variable objetivo binaria, is_influencer, para clasificar cuentas como ‚Äúinfluyentes‚Äù (1) o ‚Äúcomunes‚Äù (0). La justificaci√≥n se basa en una regla de negocio clara y defendible en el contexto de redes sociales: una cuenta se considera influyente si est√° verificada por la plataforma (authorVerified) o si posee un alto n√∫mero de seguidores (authorFollowers > 50000). Esta definici√≥n nos permite transformar un problema ambiguo en una tarea de clasificaci√≥n concreta.

¬øQu√© t√©cnicas de codificaci√≥n y vectorizaci√≥n se aplicaron?

Se utiliz√≥ un ColumnTransformer para aplicar un preprocesamiento diferenciado a cada tipo de dato, una pr√°ctica robusta y est√°ndar en la industria:
